[Capistrano](https://github.com/capistrano/capistrano/wiki/ "Capistrano") é uma ferramenta de [deploy](http://pt.wikipedia.org/wiki/Implanta%C3%A7%C3%A3o_de_software "Implantação de Software") automático, usada para facilitar a vida do programador na hora de enviar o software para o ambiente de produção.

### Multistage

Multistage é o cenário da maioria das empresas atualmente, que significa ter vários estágios entre o desenvolvimento e o sistema em produção. Na globo.com, usamos: Ambiente Local -> DEV -> QA1 -> Staging -> PROD.

Cada passagem de estágio é feita de forma automática: *one click deploy*.

### Estrutura de diretórios

O Capistrano assume uma estrutura de diretórios específica no servidor:

```bash
$ cd /var/www/super_django
$ ls
current  # Link simbólico para a versão atual
shared   # Arquivos persistentes entre deploys
releases # Histórico de versões (pastas com timestamp)
```

Dessa forma o Capistrano facilita o **rollback**. Um comando `cap rollback` faz com que ele aponte o link simbólico `current` para a versão anterior no `releases`.

### Configuração para Não-Rails

Para usar com Django ou outros frameworks, precisamos sobrescrever algumas tasks específicas do Rails no `Capfile`:

```ruby
namespace :deploy do
    task :finalize_update do
        # desabilita symlinks automáticos do rails
    end

    task :restart, :roles => :app do
        # configure seu comando de restart aqui
    end
end
```

### Conclusão

Foi fácil criar um mecanismo automático de deploy para qualquer projeto que não seja Rails. Para executar o deploy, basta digitar: `cap <<ambiente>> deploy`.
